#### 感悟

能力的提升的前提是感觉自己写的代码就是个垃圾，垃圾代码的体现

- 修改原需求时（无论是修改逻辑还是增加逻辑），牵一发而动全身
- 代码阅读性差，移植性差，复用性差
- 写出来的程序，不关注内存泄露

如何提升

- 经常重构自己之前的代码
- 通过阅读《设计模式》、《重构》等书籍，从思想上纠正不良编码风格，遵从 **开放封闭**、**高内聚低耦合**等原则进行编码
- 关注内存泄露
- 多看大佬的源代码，从中学习，进而总结， 从源码的设计或写法中 学习一些可以使用到实际项目的编码方式
